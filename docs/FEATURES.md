# 功能清单

## 1. 认证与区服

- Garmin 中国区直连
- 邮箱 + 密码登录
- 账号诊断
- 最近活动 / 设备 / 资料识别

## 2. 数据导出

- 用户资料
- 每日健康：心率、步数、睡眠、压力、Body Battery、楼层、呼吸、SpO2、HRV
- 高级健康：训练准备度、训练状态、体能年龄、跑步耐受、赛跑预测等
- 活动列表与单活动详情
- 活动 FIT / GPX
- 身体成分
- 目标 / 勋章 / 挑战 / PR
- 设备 / 闹钟 / 设置
- 装备与默认装备
- 水合与营养
- 训练计划 / 训练列表 / 日程训练
- 高尔夫
- 历史趋势

## 3. 数据导入

- 训练计划创建
- 训练计划排程
- 训练删除
- 快捷跑步训练
- 手动活动
- 体重记录
- 饮水记录
- 血压记录
- 身体成分记录
- 活动导入

## 4. AI 教练

- `morning`：晨间准备度
- `evening`：当日训练总结
- `weekly`：周总结
- `monthly`：月总结
- `alerts`：预警检测
- `plan`：自适应训练计划
- `gear`：装备寿命分析
- `confidence`：比赛信心评分

## 5. 长期记忆

- `obsidian/Coach Memory.md`
- `obsidian/Training Plan.md`
- `obsidian/Recovery Log.md`
- `obsidian/Daily/*.md`

## 6. 报告系统

- CLI 输出 JSON
- REST API 输出 JSON
- MCP 输出 JSON
- 教练报告可写入 Markdown

## 7. 负荷算法

- 准备度评分
- ACWR
- HRV / RHR / 睡眠 / Body Battery 融合
- 连续硬课提醒
- 伤病风险扣分
- 装备寿命风险

## 8. 仓库

- 导出 JSON 入 SQLite
- 活动 / 日度指标 / 设备 / 训练 / 装备统一入库
- 可做本地查询与后续分析

## 9. 指标缓存

- `data/metrics/metric_cache.json`
- 缓存每日健康摘要、活动日聚合、7 日/前 7 日/30 日趋势
- AI 教练优先读摘要和趋势，减少重复扫原始 JSON

## 10. 同步模式

- `quick`: 当日健康 + 高级健康 + 7~14 日活动，适合晨晚
- `smart`: 日常默认，健康/活动必拉，装备/训练按当天缓存跳过
- `full`: 全量类别，适合每周/月或数据不一致时
- `none`: 只用本地缓存和 SQLite，适合快速复盘
- `deep`: 深度复盘，强制 full + 至少 365 天活动回溯 + 8 周计划 + 数据覆盖清单

## 10.1 数据覆盖清单

- `data/metrics/data_inventory.json`
- 记录各类导出文件数量、最新文件、活动日期范围、最新活动、缺失健康指标
- 深度复盘时用于判断结论置信度

## 11. 复用

- 单机可直接跑
- 迁移靠复制项目目录 + `.env` + 本地 `data/` + `obsidian/`
- 不依赖云端状态才能看历史数据
